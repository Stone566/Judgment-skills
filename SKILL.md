# 决策判断模型技能

## 核心目标

帮助用户建立结构化的决策模型，通过以下步骤将模糊的判断转化为可量化的决策流程：

1. **解析决策场景** — 明确决策目标（二选一、排序、评分）
2. **评估决策属性** — 确认重要性和可逆性
3. **识别预测因子** — 引导用户找出6-8个关键变量
4. **标准化评分** — 将各因子转化为可比较的标准分
5. **计算判断得分** — 应用均等权重模型得出结果
6. **给出决策建议** — 基于阈值提供明确的行动建议

## 使用流程

### 第一步：解析决策场景

当用户提出决策需求时，首先明确：

**决策目标类型：**
- **二选一**：是/否、通过/不通过、录用/不录用
- **排序**：多个选项的优先级排列
- **评分**：对单个选项进行评价

**关键问题：**
- "您需要做什么样的决定？"
- "这个决定的重要性如何？（高/中/低）"
- "如果决定错了，能否逆转？（可逆/不可逆）"

**重要性-可逆性矩阵：**
| | 可逆 | 不可逆 |
|---|---|---|
| **高重要性** | 可以大胆尝试，快速迭代 | 必须极其谨慎，考虑最坏情况 |
| **低重要性** | 快速决定，不纠结 | 不值得花太多时间 |

### 第二步：识别预测因子

引导用户识别6-8个与决策结果明显相关的预测因子（变量）。

**选择标准（必须同时满足）：**
1. **明显相关**：因子与决策结果有清晰的逻辑关联
2. **事前可获取**：在做出决策前就能获得该信息
3. **相对客观**：可以量化或有明确的评判标准

**常见因子类型参考：**

| 类型 | 说明 | 示例 |
|------|------|------|
| 历史表现 | 过去的行为或成果 | 销售额、项目完成率、历史评分 |
| 能力指标 | 反映能力的量化数据 | 测试分数、技能等级、经验年限 |
| 匹配度 | 与需求或标准的契合程度 | 岗位匹配度、战略契合度 |
| 风险指标 | 潜在负面因素 | 违约记录、失败次数、负面评价 |
| 成本因素 | 资源投入相关 | 预算、时间成本、人力成本 |
| 增长潜力 | 未来发展空间 | 学习速度、成长轨迹、可培养性 |

**引导话术：**
- "您认为哪些因素最能预测这个决策的成功？"
- "如果是您来做这个判断，您会关注哪些方面？"
- "有没有一些明显的危险信号或积极信号？"

### 第三步：设计评分标准

为每个因子设计1-5分或1-10分的评分标准。

**评分标准模板：**

```
因子：[因子名称]
含义：[具体指什么]
评分标准：
- 5分：[优秀/完全符合/极低风险的描述]
- 4分：[良好/基本符合/低风险的描述]
- 3分：[一般/部分符合/中等风险的描述]
- 2分：[较差/不太符合/高风险的描述]
- 1分：[很差/完全不符合/极高风险的描述]
```

**注意事项：**
- 每个分数档位必须有明确、可区分的描述
- 避免中间档位过于宽泛
- 尽量使用客观指标而非主观感受

### 第四步：收集数据与评分

**情况A：用户有历史数据**
- 询问每个因子的具体数值
- 使用标准化公式转换为Z分数

**情况B：用户没有历史数据**
- 直接进行1-5分的主观评分
- 建议用户基于直觉快速打分，不要过度思考

**标准化公式（当有时历史数据时）：**
```
z_i = (x_i - μ_i) / σ_i

其中：
- x_i = 原始值
- μ_i = 历史平均值
- σ_i = 历史标准差
- z_i = 标准分数（通常范围-3到+3）
```

### 第五步：计算判断得分

**均等权重模型公式：**
```
Ŷ = (z_1 + z_2 + ... + z_n) / n

或对于1-5分评分：
Ŷ = (score_1 + score_2 + ... + score_n) / n
```

**特点：**
- 所有因子权重相等
- 简单、透明、不易过拟合
- 卡尼曼研究表明：均等权重往往优于专家赋权

### 第六步：设定决策阈值

根据决策类型设定阈值：

**二分类决策阈值：**
| 阈值位置 | 适用场景 |
|---------|---------|
| 高于平均值（>0） | 需要积极筛选（如投资机会） |
| 平均值（=0） | 平衡型决策 |
| 低于平均值（<0） | 保守型决策（如贷款审批） |

**评分决策参考：**
- 平均分 ≥ 4.0：强烈推荐
- 平均分 3.0-4.0：可以考虑
- 平均分 2.0-3.0：需谨慎
- 平均分 < 2.0：不建议

### 第七步：生成判断报告

使用以下模板输出最终报告：

```markdown
## 判断报告：[决策场景]

### 预测因子评分
| 因子 | 权重 | 原始值 | 标准化分数 |
|------|------|--------|-----------|
| [因子1] | 均等 | [值] | [分数] |
| [因子2] | 均等 | [值] | [分数] |
| ... | ... | ... | ... |

### 计算结果
- **判断得分**：[X.XX]
- **决策阈值**：[X.XX]
- **建议决策**：[通过/不通过/需进一步评估]

### 关键洞察
- **最强正向因子**：[因子X] — [具体说明]
- **最强负向因子**：[因子Y] — [具体说明]
- **风险提示**：[如有]

### 模型说明
本判断基于卡尼曼《噪声》中的机械判断模型，消除了人类直觉中的随机噪声。

⚠️ **重要提醒**：
- 如有明确的"断腿信息"（模型无法考虑的极端特殊情况），请人工 override
- 模型提供的是参考，最终决策权在您
- 对于不可逆的重要决策，建议增加审核步骤

### 检查清单
- [ ] 所有因子在决策前都可获取
- [ ] 评分标准相对客观
- [ ] 考虑了最坏情况
- [ ] 确认没有遗漏明显的"断腿信息"
```

## 特殊处理

### 断腿信息（Override 规则）

如果存在以下情况，模型结果仅供参考，需要人工 override：

- **极端负面信息**：如欺诈记录、严重违约、道德问题
- **极端正面信息**：如行业顶尖专家、独家资源、战略级合作
- **模型未考虑的约束**：如合规要求、政策变化、突发危机

### 多选项排序

当需要对多个选项排序时：
1. 对每个选项分别运行模型
2. 按判断得分从高到低排序
3. 输出排序结果和得分差异分析

### 敏感性分析

可以向用户解释：
- 改变某个因子的评分会对结果产生什么影响
- 哪些因子对结果最敏感
- 在哪些情况下建议重新评估

## 输出原则

1. **透明**：展示所有计算过程和假设
2. **简洁**：避免过度复杂的数学表达
3. **可操作**：给出明确的下一步建议
4. **谦逊**：明确模型的局限性和适用边界

## 示例对话

**用户**："帮我决定是否录用这个产品经理候选人"

**Claude**：
> 好的，我来帮您建立一个结构化的录用决策模型。
>
> **第一步：明确决策属性**
> - 决策类型：二选一（录用/不录用）
> - 重要性：高（招聘错误成本高）
> - 可逆性：部分可逆（可以试用期后解雇，但有成本）
>
> **第二步：识别预测因子**
> 基于产品经理岗位，我建议关注以下因子（您可以根据实际情况调整）：
> 1. 产品经验年限
> 2. 案例分析/实操表现
> 3. 沟通协作能力
> 4. 行业匹配度
> 5. 学习能力/潜力
> 6. 薪资期望匹配度
>
> 您觉得这个框架合适吗？需要添加或删除哪些因子？

（继续引导用户完成评分和计算）
